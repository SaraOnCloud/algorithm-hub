The Longest Increasing Subsequence (LIS) problem asks for the longest subsequence of numbers (not necessarily contiguous) that is strictly increasing.

This interactive playground lets you explore two standard approaches:

  • O(n²) Dynamic Programming: For each position i, compute dp[i] = length of LIS ending at i by scanning all j < i.
  • O(n log n) Patience (Greedy + Binary Search): Maintain pile tops (minimal tail for each subsequence length). Uses a binary search to place each element, mirroring the card game patience.

Hit Play and watch the algorithm build the solution step-by-step. Switch modes anytime (resets progress).

Complexity

  • DP: Time O(n²), Space O(n)
  • Patience: Time O(n log n), Space O(n)
  • Both reconstructions take linear extra work.

Patience gives length efficiently; to reconstruct a path we maintain parent pointers.

DP Pseudocode

dp[i] = 1
prev[i] = -1
for i in [0..n-1]:
  for j in [0..i-1]:
    if a[j] < a[i] and dp[j] + 1 > dp[i]:
       dp[i] = dp[j] + 1
       prev[i] = j
answer = max(dp)
reconstruct by following prev from argmax

Patience Pseudocode

tails = []  # minimal tail for each length
parent = [] # for reconstruction
for i, x in enumerate(a):
  k = lower_bound(tails, x)
  if k == len(tails): tails.append(x)
  else: tails[k] = x
  # link parent to previous tail index at length k-1
length = len(tails)
reconstruct by walking stored parent pointers
